## Design Context

### Users
Single user — the developer themselves, an active swing trader on Borsa Istanbul (BIST). Uses the app primarily on mobile during market hours (10:00-18:00 Istanbul time) to check signals, manage positions, and execute trades. Evening usage on desktop for deeper portfolio review and analysis. Needs to glance at data quickly and act fast — every second counts during market hours.

### Brand Personality
**Fast, bold, energetic.** This is a personal fintech command center, not a bank portal. It should feel like a high-performance cockpit — responsive, confident, and action-oriented. The interface should convey momentum and control without being cluttered.

Three words: **driven, precise, alive.**

### Aesthetic Direction
- **Tone**: Fintech startup energy meets trading desk efficiency. Think Robinhood/Midas simplicity but with more data density for an experienced trader. Clean enough to parse at a glance on mobile, bold enough to feel exciting.
- **Theme**: Dark mode only. This is a trading dashboard used in focused sessions — dark reduces eye strain and makes numbers pop.
- **References**: Robinhood (clean mobile UX, bold typography, confident use of space), Midas (Turkish fintech, mobile-first, modern feel).
- **Anti-references**: Bloomberg Terminal (too dense for mobile), generic admin dashboards (gray cards with side-stripe borders), anything that looks like "AI made this" (cyan-on-dark, purple gradients, neon glows).
- **Language**: All UI text is Turkish. Labels should be concise and direct — no fluff.

### Color Direction
The current palette (deep navy + red/green) is too close to the generic "AI dark dashboard" look. New palette should:
- Keep red/green semantics for loss/profit (this is universal in finance) but find more distinctive, less saturated versions
- Replace the navy (#1a1a2e) background with something warmer or more characterful — tinted toward the brand accent
- Avoid pure black (#000) and pure white (#fff)
- Avoid cyan, neon, purple-to-blue gradients
- Use OKLCH for perceptual uniformity

### Design Principles
1. **Glanceable over decorative** — Every element must earn its place. A trader checking their phone during market hours needs to read the situation in under 2 seconds.
2. **Mobile-first, desktop-enhanced** — The primary surface is a phone screen. Desktop gets more data, not a different experience.
3. **Bold hierarchy, not bold everything** — Use size and weight contrast aggressively for the most important numbers (P&L, price, score). Let supporting data recede.
4. **Alive, not noisy** — Subtle motion and color shifts to convey state changes (profit growing, signal strength), but never gratuitous animation.
5. **One tool, one person** — No need for onboarding, help text, or multi-user patterns. Every pixel serves the single trader who built it.
